# LiDAR Touch Controller

Проект LiDAR Touch Controller позволяет использовать лидар LD-M1S как сенсорный экран, отслеживая прикосновения к поверхности и управляя курсором мыши.

## 📋 Описание

Приложение превращает лидар LD-M1S в сенсорную панель: сканирует поверхность, обнаруживает касания по отклонениям от фоновой карты и преобразует координаты касания в движения курсора мыши.

### Возможности:
- ✅ Калибровка фона помещения (создание фоновой карты)
- ✅ Фильтрация шума и постоянных помех (персистентное хранение)
- ✅ Мгновенная детекция касаний (FastTouchDetector, lookup-таблица)
- ✅ Управление курсором мыши (перемещение, drag, выделение областей)
- ✅ Калибровка 4 углов рабочей зоны с перспективным преобразованием (гомография)
- ✅ **Кроссплатформенное управление мышью** (Windows: Win32, Linux: ydotool/Xlib/Wayland)
- ✅ **Автоматическая установка зависимостей** при первом запуске
- ✅ Графический интерфейс с визуализацией в реальном времени
- ✅ Поддержка **Astra Linux** (Fly DE, X11, Wayland)

## Архитектура проекта

### Уровень 1: Презентация (GUI)
- `app.py` — главный фасад, связывает все компоненты
- `gui_builder.py` — построение Tkinter интерфейса (кнопки, статусы, лог)
- `plot_manager.py` — визуализация данных на matplotlib (полярный график)

### Уровень 2: Бизнес-логика (Controller)
- `app_controller.py` — центральная оркестрация:
  - Инициализация лидара и мыши
  - Запуск цикла детекции
  - Обработка callbacks от GUI и SDK
  - Проверка готовности системы

### Уровень 3: Обработка данных (Business Modules)

#### Калибровка (`calibration/`)
- `calibrator.py` — оркестрация процесса калибровки
- `data_handlers.py` — построение фоновой карты (медиана + интерполяция)
- `file_io.py` — сохранение/загрузка `.npz`

#### Детекция касаний (`detection/`)
- `touch_detector.py` — полный конвейер детекции (8 шагов фильтрации)
- `fast_detector.py` — оптимизированный детектор для мыши (LUT + векторизация)
- `comparators.py` — функции сравнения сканов и отклонений
- `threshold_config.py` — пресеты порогов (default/sensitive/robust)

#### Фильтрация шума (`noise_filter/`)
- `noise_profiler.py` — сбор статистики ложных срабатываний
- `noise_filter.py` — фильтрация по профилю шума
- `corner_calibration.py` — калибровка 4 углов проекции

#### Постоянный шум (`persistent_noise_filter/`)
- `persistent_noise_manager.py` — долгосрочное хранение шумовых точек (JSON)

### Уровень 4: Аппаратный слой (Hardware)

#### Лидар (`lidar_sdk/`)
- `lidar_m1.py` — драйвер лидара (чтение пакетов, сбор сканов)
- `connection.py` — TCP соединение (192.168.0.7:25168)
- `protocol.py` — парсинг пакетов, CRC8 валидация
- `models.py` — модели данных (Point, Node, DataPackM1)

#### Управление мышью (`mouse_controller/`)
- `mouse_controller.py` — мультибэкенд контроллер:
  - Windows: Win32 API
  - Linux: ydotool → Xlib → wayland-automation (fallback цепочка)
  - Гомография: проекция LiDAR → экран

### Уровень 5: Системная настройка (Setup)
- `system_setup.py` — автоматическая установка зависимостей:
  - Python пакеты (pip)
  - Системные утилиты (apt)
  - Права доступа (udev правила)

## 📊 Поток данных

- LiDAR (TCP) → lidar_sdk (парсинг пакетов → Point)
- lidar_sdk → app_controller (_detection_loop)
- app_controller → detection/fast_detector (касания)
- detection → noise_filter (фильтрация ложных)
- detection → mouse_controller (движение курсора)
- app_controller → calibration (фоновая карта)
- app_controller → noise_filter (профиль шума)
- app_controller → corner_calibration (углы проекции)

## 🖥️ Бэкенды управления мышью (Linux)

**Приоритет 1: ydotool**
- Платформа: X11 + Wayland
- Требования: `/dev/uinput`, демон `ydotoold`
- Команды: `mousemove`, `mousedown`, `mouseup`

**Приоритет 2: Xlib**
- Платформа: X11 только
- Требования: `python-xlib`, переменная `$DISPLAY`
- Команды: `warp_pointer`, `fake_input`

**Приоритет 3: wayland-automation**
- Платформа: Wayland (wlroots)
- Требования: `wayland-automation` пакет
- Ограничения: поддержка только wlroots композитеров

    'connection_timeout': 5.0

## 🖱️ Использование

### 1. Подключение к лидару
При запуске приложение автоматически подключается к лидару. Статус отображается в панели «Status».

### 2. Создание фоновой карты
- Убедитесь, что перед лидаром нет объектов
- Нажмите **«Create Background Map»**
- Приложение соберёт 10 сканов и сохранит `background.npz`

### 3. Профилирование шума (опционально)
- Нажмите **«Profile Noise»**
- Не касайтесь поверхности в течение 10 секунд
- Результат сохраняется в `noise_profile.json`

### 4. Калибровка углов проекции
- Нажмите **«Calibrate All Corners»** — калибровка всех 4 углов по очереди
- Или используйте отдельные кнопки для каждого угла
- Перед каждым шагом поднесите палец к соответствующему углу рабочей зоны
- 5-секундный обратный отсчёт перед каждым этапом
- Результат сохраняется в `projection_corners.json`

### 5. Включение управления мышью
- Нажмите **«Start Mouse Control»** (toggle-кнопка)
- При касании поверхности:
  - **ЛКМ зажимается** при начале касания
  - Курсор двигается за пальцем (drag / выделение области)
  - **ЛКМ отпускается** при убирании пальца
- Мёртвая зона **50px** — курсор двигается только при смещении > 50px от точки касания

## 🛠️ Архитектура обработки

### Детекция касаний (FastTouchDetector)
1. Lookup-таблица 0–359° — O(1) доступ к фоновому расстоянию
2. Векторизованный поиск кандидатов с отклонением ≥ 50 мм
3. Группировка по угловому окну (1.0°)
4. Центр массы лучшего кластера — циркулярное среднее

### Управление мышью (мультибэкенд)

| Платформа | Бэкенд | Приоритет |
|-----------|--------|-----------|
| Windows | Win32 API | Единственный |
| Linux | ydotool | 1 (универсальный, X11 + Wayland) |
| Linux | Xlib | 2 (только X11) |
| Linux | wayland-automation | 3 (только Wayland wlroots) |

**Алгоритм работы:**
1. Гомография (DLT): 4 угла LiDAR (мм) → 4 угла экрана (px)
2. Мгновенное перемещение: `SetCursorPos` (Win) / `ydotool mousemove` (Linux)
3. Касание → ЛКМ DOWN, движение → drag, отпускание → ЛКМ UP
4. Мёртвая зона: курсор фиксируется пока смещение < 50px

### Оптимизации
- Цикл детекции обрабатывает только **новые** сканы (scan_counter)
- Без `time.sleep()` в рабочем пути
- Без EMA-сглаживания (мгновенная реакция)
- Векторизованные операции numpy для фильтрации

## 🐧 Особенности для Astra Linux

### Автоматическая настройка
При первом запуске с `sudo` приложение автоматически:
- Устанавливает `ydotool` через apt
- Создаёт правило udev для `/dev/uinput`
- Запускает демон `ydotoold`

### Ручная настройка (если автоустановка не сработала)

```bash
# 1. Установить ydotool
sudo apt install ydotool

# 2. Дать права на uinput
sudo chmod 666 /dev/uinput

# 3. Создать постоянное правило
echo 'KERNEL=="uinput", MODE="0666"' | sudo tee /etc/udev/rules.d/99-uinput.rules
sudo udevadm control --reload-rules

# 4. Запустить демон
sudo ydotoold &

# 5. Запустить приложение
python3 app.py
```

### Диагностика
```bash
# Проверка ydotool
which ydotool

# Проверка прав доступа
ls -l /dev/uinput

# Проверка сессии
echo $DISPLAY
echo $WAYLAND_DISPLAY

# Тест движения курсора
sudo ydotool mousemove 500 500
```
